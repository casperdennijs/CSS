#1 - Scroll driven animation

Scroll driven animation

Scroll driven animation betekent dat een animatie wordt gestuurd door de positie van de gebruiker op de pagina tijdens het scrollen. In plaats van op een tijdlijn te spelen (zoals een gewone CSS-animatie), wordt de animatie gesynchroniseerd met de scrollpositie.


@scroll-timeline

Scroll-driven animatie is mogelijk dankzij een nieuwe API: @scroll-timeline en de eigenschap animation-timeline



Einde pagina

section > div {
    height: 10px;
    width: 0%;
    background-color: green;
    position: sticky;
    top: -1rem;
    left: 0;
    animation: progress auto linear;
    animation-timeline: --fullPageScroll;
    margin-bottom: -10px;
}

section {
    scroll-timeline: --fullPageScroll block;
}

@keyframes progress {
    to { width: 100% }
}

#2 - Anchor positioning API

Anchor positioning API

De CSS Anchor Positioning API is een nieuwe webtechniek waarmee je elementen op een pagina kunt positioneren ten opzichte van andere elementen (de zogenaamde “anchors”). Dit biedt meer controle en flexibiliteit bij het maken van dynamische layouts, zonder dat je JavaScript nodig hebt.


Met deze API kun je bijvoorbeeld een tooltip, dropdown of label relatief aan een ander element positioneren, zelfs als dat element op een andere plek in de DOM staat. Je gebruikt hiervoor het anchor-name en anchor() in je CSS.



Momenteel gebruik ik de anchor functie om vanuit de top en left hoeken het ::after element te verplaatsen in hoek rechtsonder

top: anchor(bottom);
left: anchor(right); 
                    
div {
    height: 100px;
    width: 100px;
    background-color: yellow;
    margin: 50px;
    anchor-name: --anchor-el;
}

div::after {
    position-anchor: --anchor-el;
    height: 50px;
    width: 50px;
    background-color: blue;
    display: block;
    content: "";
    position: absolute;
    top: anchor(bottom);
    left: anchor(right);
}

#3 - :has() selector

:has() selector

Met de :has selector kan je vanaf nu aangeven om een element aan te passen op basis van een criteria die binnen het element voldaan wordt. Als voorbeeld gebruik ik een formulier waarbij we checken of de algemene voorwaarden checkbox aangevinkt is. Wanneer dit zo is laten we namelijk pas de voltooi button zien.



Door de :has selector te gebruiken hebben we geen verdere JavaScript nodig om de Voltooien button te tonen en kunnen we doormiddel van CSS controleren of iets is ingevuld.


Verder heb ik de :has selector() over deze hele website gebruikt om te bepalen of je de preview of code moet zien per categorie. Dit doe ik doormiddel van een fieldset met radio buttons waarin ik kijk welke radio button met welke id checked is om zo de juiste content te tonen.

/* Voorbeeld in de preview */
form button {
    display: none;
}

form:has(input[type="checkbox"]:checked) button {
    display: block;
}



/* Preview & code tabs */
main > section > section {
    display: none;
    background-color: #202020;
    color: #fff;
    width: 100%;
    border-top: 2px solid #202020;
    padding: 2rem;
    border-radius: 0 0 10px 10px;
    overflow: hidden;
    box-shadow: 0px 10px 5px -2px rgba(0,0,0,0.25);
}

main > section:has(input#preview:checked) > section#preview {
    display: block;
}

main > section:has(input#code:checked) > section#code {
    display: block;
}
